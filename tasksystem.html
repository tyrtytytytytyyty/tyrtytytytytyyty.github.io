Task System

GIF coming soon

A modular objective framework that drives the round-by-round gameplay loop. Tasks are defined as ModuleScripts — each declares its trigger conditions, validation logic, and reward hook. The TaskManager replicates state to clients, listens for completion signals, and resolves rewards atomically.

Architected the system and shipped the manager plus the first half-dozen task types. Key design decision: tasks own their own validation logic instead of relying on the manager. That keeps the manager dumb and the task library extensible — adding a new objective is a single new ModuleScript, not a refactor.
